
# Stakeholder Notes

## Purpose
This file records initial notes from stakeholders for the Student Task Management System. These notes are used as input for the AI skills, especially inception, elicitation, specification, prioritization, and validation.

## Case Context
Student Task Management System is a campus application used by lecturers, students, and administrators to manage assignments, submissions, grading, deadlines, and reporting.

## Lecturer Notes
- Lecturers need to create assignments for specific courses.
- Each assignment should include a title, description, course, deadline, and submission instructions.
- Lecturers need to set and update assignment deadlines.
- Lecturers need to see which students have submitted, have not submitted, submitted late, or already received grades.
- Lecturers need to open submitted files for review.
- Lecturers need to enter grades and written feedback for submitted assignments.
- Lecturers need a clear submission list per assignment because manual tracking can cause missed submissions.

## Student Notes
- Students need one place to view active assignments for their enrolled courses.
- Students need to see assignment deadlines clearly.
- Students need to upload assignment files.
- Students need confirmation after uploading a file so they know the submission was received.
- Students need to monitor assignment status, such as Not Submitted, Submitted, Late, Graded, or Returned.
- Students need to view grades and feedback after lecturers finish assessment.

## Administrator Notes
- Administrators need to manage lecturer and student accounts.
- Administrators need to manage courses.
- Administrators need to manage enrollment relationships between students, lecturers, and courses.
- Administrators need to configure accepted file types for assignment submission.
- Administrators need to configure maximum file size for submissions.
- Administrators need to configure late submission policy.
- Administrators need basic assignment activity reports.

## Notes for AI Use
- These notes are stakeholder input, not final requirements.
- Any missing information must be marked as `OPEN QUESTION`.
- Any assumption that is not directly stated here must be marked as `ASSUMPTION`.
- Unsupported features such as chat, plagiarism detection, payment, and video conferencing must not be added as confirmed requirements.
